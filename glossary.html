LLM부터 MCP까지 — 자주 등장하는 21가지 AI 용어를 일상 비유로 설명하고, 헷갈리는 개념 비교와 자주 묻는 질문까지 한 곳에 담았습니다.

이 페이지 목차 꼭 알아야 할 AI 용어 헷갈리는 개념 비교 자주 묻는 질문 안전하고 똑똑하게 쓰기 다음 단계

꼭 알아야 할 AI 용어

처음 보면 낯설지만, 비유 하나면 바로 이해됩니다. 실제로 AI를 쓸 때 자주 등장하는 핵심 용어들만 추렸습니다.

① 기본 개념

LLM(거대 언어 모델) Large Language Model

수천억 개의 문장을 학습해 다음에 올 말을 예측하도록 훈련된 AI 모델. Claude, ChatGPT, Gemini 모두 LLM입니다.

인터넷 전체 책을 읽은 뒤 "다음 단어는 뭘까?" 를 극도로 잘하는 시스템.

프롬프트 Prompt

AI에게 보내는 입력 텍스트. 질문, 지시, 역할, 예시 등 AI에게 전달하는 모든 말을 포함합니다.

직원에게 주는 업무 지시서. 구체적일수록 결과물이 좋습니다.

토큰 Token

AI가 텍스트를 처리하는 최소 단위. 영어는 단어 하나가 보통 1토큰, 한국어는 글자 1~2개가 1토큰입니다. 요금과 처리 길이의 기준입니다. 참고로 한국어는 같은 내용의 영어보다 토큰을 더 많이 쓰는 경향이 있어, 비용·길이에 영향을 줍니다.

레고 블록 하나. AI는 문장을 블록 단위로 분해해서 읽고 씁니다.

컨텍스트 윈도우 Context Window

AI가 한 번에 기억할 수 있는 최대 토큰 수. 이 범위를 넘으면 앞 내용을 잊어버립니다. 최신 Claude 모델은 약 200K 토큰까지 다룹니다(모델에 따라 다름).

단기기억 노트의 크기. 노트가 꽉 차면 가장 오래된 메모부터 지워집니다.

환각(할루시네이션) Hallucination

AI가 사실이 아닌 내용을 자신 있게 말하는 현상. 존재하지 않는 논문, 틀린 날짜, 가짜 인용문 등이 대표 사례입니다.

시험에서 모르는 문제에 아는 척 그럴듯한 답을 써내는 학생. 자신 있어 보이지만 틀릴 수 있습니다.

지식 컷오프 Knowledge Cutoff

AI 모델이 학습한 데이터의 마지막 날짜. 그 이후에 일어난 사건은 알지 못합니다. 단, 웹검색이 연결된 경우 최신 정보를 보충할 수 있습니다.

특정 날짜에 인쇄된 백과사전. 그날 이후 뉴스는 실려 있지 않습니다.
② 작동 원리

파라미터 Parameter

AI 모델이 학습으로 조정된 내부 숫자 가중치. 모델 규모를 나타낼 때 씁니다. 대형 모델은 수천억~수조 개의 파라미터를 갖는다고 알려져 있습니다(정확한 수치는 비공개인 경우가 많음). 많을수록 복잡한 패턴을 학습합니다.

뇌의 시냅스 연결 강도. 수가 많을수록 더 많은 패턴을 기억합니다.

학습 / 훈련 Training

대규모 텍스트 데이터로 모델의 파라미터를 수없이 조정하는 과정. 몇 주에서 몇 달이 걸리고, 수백억 원의 컴퓨팅 비용이 들 수 있습니다.

수천만 권의 책을 읽히며 반복 시험을 치러 틀릴 때마다 교정하는 과정.

파인튜닝 Fine-tuning

기존 훈련된 모델에 특정 도메인(의료·법률·고객상담)의 데이터를 추가 학습시켜 전문화하는 것.

이미 졸업한 의사에게 외과 전공의 수련을 더 시키는 것.

임베딩 Embedding

단어나 문장의 의미를 숫자 벡터(좌표)로 변환한 것. "사과"와 "과일"은 벡터 공간에서 가까운 위치에 있어 AI가 유사도를 계산할 수 있습니다.

세계 지도에서 비슷한 기후의 도시들은 가까이 위치하는 것처럼, 비슷한 의미의 단어는 좌표상 가깝습니다.

Temperature(창의성 조절) Temperature

AI의 다음 단어 선택 무작위성을 조절하는 수치(보통 0~2). 낮으면 예측 가능하고 일관되며, 높으면 창의적이지만 뜬금없어질 수 있습니다.

커피의 진하기 조절. 낮으면 아메리카노처럼 정확하고, 높으면 여러 재료를 섞은 창작 음료처럼 독특합니다.

추론 모델 Reasoning Model

답을 바로 내놓지 않고 "생각하는 과정(Chain of Thought)"을 거쳐 복잡한 수학·논리 문제를 푸는 데 특화된 모델. OpenAI o-시리즈, Claude의 확장 사고(Extended Thinking) 모드 등.

계산기를 꺼내 하나씩 풀어가는 학생 vs. 바로 직감으로 답 쓰는 학생. 어려운 문제일수록 차이가 납니다.
③ 활용·개발 용어

RAG(검색 증강 생성) Retrieval-Augmented Generation

AI가 답변하기 전 외부 문서(사내 DB, PDF, 웹 등)를 검색해 관련 내용을 가져온 뒤 생성하는 방식. 최신 정보나 내부 데이터를 활용할 수 있습니다.

오픈북 시험. 시험지를 받은 뒤 교재를 찾아보고 답을 쓰는 것.

멀티모달 Multimodal

텍스트뿐 아니라 이미지, 오디오, 동영상 등 여러 형태의 입력을 동시에 처리하는 AI. Claude 3·GPT-4o가 대표 예입니다.

글, 사진, 말을 모두 이해하는 만능 통역사.

시스템 프롬프트 System Prompt

사용자가 보내는 첫 메시지 이전에 AI의 역할·행동 규칙을 미리 지정하는 숨겨진 지시문. 서비스 운영자가 설정합니다.

직원이 손님을 만나기 전 받는 사내 교육. "항상 친절하게, 경쟁사 이름은 언급 금지" 같은 규칙입니다.

에이전트 Agent

목표를 받으면 스스로 계획을 세우고, 도구를 호출하고, 결과를 평가하며 반복 실행하는 AI. 단순 채팅과 달리 다단계 작업을 자율 수행합니다.

심부름꾼이 아니라 프로젝트 매니저. 지시 하나에 여러 단계를 알아서 해결합니다.

MCP Model Context Protocol

AI가 외부 도구(파일 시스템, 데이터베이스, API 등)와 통신하는 표준 규격. Anthropic이 제안, 현재 업계 표준으로 자리잡고 있습니다.

USB 표준 규격. 어떤 기기든 USB 포트만 있으면 꽂아 쓸 수 있듯, MCP가 있으면 어떤 도구든 AI에 연결 가능합니다.

API Application Programming Interface

소프트웨어끼리 서로 통신하는 규칙과 접점. AI API를 쓰면 내 앱이나 서비스 안에 AI 기능을 직접 심을 수 있습니다.

식당의 주문 창구. 주방에 직접 들어가지 않아도, 정해진 메뉴판(API)으로 원하는 요리를 받을 수 있습니다.

함수 호출 Function Calling

AI가 답변 도중 "지금 날씨 검색" 같은 외부 함수를 호출하고 결과를 받아 답변에 반영하는 기능. 에이전트의 핵심 기술입니다.

비서가 답변 중 잠깐 전화로 확인하고 돌아와 최신 정보로 보고하는 것.

토큰 비용 / 요금 Token Pricing

API를 쓸 때 입력·출력 토큰 수에 따라 과금됩니다. 일반 사용자는 월정액 구독, 개발자는 API 사용량 기반 요금을 냅니다.

전화 통화처럼 말한 시간만큼 요금이 부과됩니다. 길게 쓸수록, 많이 쓸수록 비용이 올라갑니다.

오픈소스 vs 폐쇄형 모델 Open-source vs Closed-source

오픈소스(Llama, Mistral 등)는 모델 가중치를 공개해 누구나 수정·배포 가능. 폐쇄형(Claude, GPT-4 등)은 API로만 접근 가능하고 내부는 비공개입니다.

요리 레시피를 공개(오픈소스) vs. 비밀 레시피로 판매(폐쇄형).

주요 AI 비서 한눈에 비교

Claude (Anthropic) 긴 문서 분석·추론·정직성 중시. 한국어 품질 우수. 컨텍스트 윈도우 최대 200K.
ChatGPT (OpenAI) 가장 넓은 생태계, 이미지 생성·음성·웹검색 통합. Custom GPTs 플러그인 풍부.
Gemini (Google) Google 서비스 연동(Gmail·Docs·Drive). 멀티모달 능력 강함. 무료 플랜 넉넉.
Llama / 오픈소스 가중치 공개, 로컬 실행 가능. 데이터 보안·비용 절감 필요 시 선택.

자주 묻는 질문

실제로 가장 많이 들어오는 질문들을 모았습니다. 궁금한 항목을 눌러 답변을 확인하세요.

AI 비서는 무료로 쓸 수 있나요?

네, 주요 AI 비서(Claude, ChatGPT, Gemini)는 모두 무료 플랜을 제공합니다. 무료로도 충분히 유용하게 사용할 수 있습니다.

다만 무료 플랜은 하루 사용량 제한, 가장 강력한 모델 접근 제한, 응답 속도 저하 등이 있을 수 있습니다. 업무에 자주 쓴다면 월 20~30달러 수준의 유료 플랜이 훨씬 쾌적합니다.

어떤 AI를 골라야 하나요? (Claude vs ChatGPT vs Gemini)

목적에 따라 선택이 달라집니다.

  • Claude — 긴 문서 분석, 정교한 글쓰기, 정직하고 안전한 대화, 한국어 품질에 강점
  • ChatGPT — 다양한 플러그인, 이미지 생성(DALL-E), 음성 대화, 코드 실행 환경이 필요할 때
  • Gemini — Gmail·구글 드라이브 등 구글 서비스와 깊이 연동하고 싶을 때

처음이라면 무료로 모두 써보고 가장 잘 맞는 것을 선택하세요. 각자 잘하는 영역이 달라 두 가지를 함께 쓰는 사람도 많습니다.

AI가 하는 말은 믿어도 되나요? (환각 문제)

AI는 사실처럼 들리지만 틀린 정보(환각)를 말할 수 있습니다. 특히 다음 상황에서 주의하세요.

  • 구체적인 날짜, 통계, 논문 인용, 법률 조항
  • 잘 알려지지 않은 인물이나 최신 사건
  • 계산이 필요한 수치

중요한 정보는 반드시 공식 출처에서 사실 확인하세요. AI를 "완벽한 정답지"가 아닌 "초안 작성 도우미"로 생각하면 올바르게 활용할 수 있습니다.

내가 입력한 내용이 AI 학습에 쓰이나요? (개인정보·프라이버시)

서비스마다 정책이 다르지만, 일반적으로 다음을 기억하세요.

  • 대부분의 서비스는 설정에서 대화 기록 저장·학습 활용을 끌 수 있습니다.
  • 유료 API를 통한 접근은 대체로 학습에 사용되지 않습니다.
  • 주민등록번호, 카드번호, 비밀번호, 타인의 개인정보는 절대 입력하지 마세요.

각 서비스의 개인정보처리방침과 앱 설정을 직접 확인하는 것을 권장합니다.

한국어도 잘 되나요?

네, 최신 주요 AI 모델들은 한국어를 매우 잘 처리합니다. Claude, ChatGPT-4o, Gemini 모두 한국어 입출력을 자연스럽게 지원합니다.

원하는 언어로 답해달라고 명시하면 더 일관됩니다. 예: "항상 한국어로 답해줘" 또는 "답변은 한국어로 작성해줘".

영어보다 한국어가 약간 느리거나 토큰을 더 쓰는 경우가 있지만, 품질 차이는 일상 사용에서 크게 느껴지지 않습니다.

환각(틀린 답)을 줄이는 방법은?

몇 가지 습관으로 환각을 크게 줄일 수 있습니다.

  • 출처를 함께 제공하라고 요청 — "답변에 출처 URL도 알려줘"
  • 모르면 모른다고 해줘 — "확실하지 않으면 그렇다고 알려줘"
  • 관련 문서를 첨부 — AI에게 읽을 자료를 주면 추측 없이 문서 기반으로 답변
  • 중요 정보는 검증 — 수치·법률·의학 정보는 공식 사이트에서 재확인
  • 추론 모델 활용 — 복잡한 계산·논리 문제는 OpenAI o-시리즈, Claude 확장 사고(Extended Thinking) 모드 등 사용
유료 결제는 그만한 가치가 있나요?

업무에 자주 쓴다면 충분히 가치 있습니다. 주요 혜택은 다음과 같습니다.

  • 더 강력한 최신 모델 사용 가능
  • 하루 사용량 제한 없이 쓸 수 있는 여유
  • 더 긴 컨텍스트(더 많은 내용을 한 번에 처리)
  • 이미지 생성, 파일 업로드, 코드 실행 등 추가 기능

반대로 가끔 간단한 글쓰기 도움 정도라면 무료 플랜으로 충분할 수 있습니다. 무료로 먼저 써보고, 한계를 느낄 때 업그레이드하는 방법을 추천합니다.

AI가 내 일자리를 대체하나요?

반복적·규칙적인 작업은 자동화될 가능성이 높지만, AI가 모든 일을 대체하지는 않습니다.

  • 창의적 판단, 감성적 소통, 복잡한 윤리적 결정은 여전히 사람이 필요합니다.
  • AI를 잘 활용하는 사람이 그렇지 않은 사람보다 경쟁력이 높아지는 추세입니다.
  • 많은 직군에서 AI는 대체보다 보조(Copilot) 역할을 합니다.

지금 당장 두려워하기보다, AI를 도구로 배워 내 업무 효율을 높이는 쪽이 현명합니다.

그림 · 이미지도 만들 수 있나요?

네, 가능합니다. 다만 텍스트 생성 AI와 이미지 생성 AI는 별개입니다.

  • ChatGPT Plus — DALL-E 3 이미지 생성 내장, 대화 중 이미지 제작 가능
  • Gemini — Imagen 연동 이미지 생성 지원
  • Midjourney / Adobe Firefly / Stable Diffusion — 이미지 특화 전문 도구
  • Claude — 이미지 생성은 현재 지원하지 않으나 이미지 분석은 가능
최신 정보도 아나요? (지식 컷오프 · 웹검색)

기본적으로 AI 모델에는 학습 데이터 마감일(지식 컷오프)이 있어, 그 이후 사건은 모릅니다.

단, 아래 경우에는 최신 정보를 얻을 수 있습니다.

  • ChatGPT — 웹검색(Browse) 기능 켜기
  • Claude — 웹검색 도구 활성화 (지원 여부 확인 필요)
  • Gemini — Google 검색 연동으로 실시간 정보 접근

웹검색이 연결돼 있어도, AI가 검색 결과를 그대로 전달하는 것이 아니라 요약·해석하는 것이므로 중요한 내용은 직접 확인하세요.

코딩을 몰라도 쓸 수 있나요?

물론입니다! AI 비서의 장점 중 하나가 코딩 없이도 복잡한 작업을 할 수 있다는 점입니다.

  • 엑셀 수식, 문서 작성, 번역, 이메일 초안 — 코딩 전혀 불필요
  • "파이썬으로 엑셀 정리하는 코드 짜줘"처럼 요청하면 코드를 대신 써줍니다
  • Cowork 등 에이전트 도구는 자연어로 파일 작업을 수행

코딩을 모른다고 주눅 들지 마세요. AI는 기술자보다 좋은 질문을 하는 사람이 더 잘 씁니다.

모바일에서도 되나요?

네, 주요 AI 비서는 모두 모바일 앱을 제공합니다.

  • Claude — iOS / Android 앱 출시, 앱스토어에서 다운로드
  • ChatGPT — iOS / Android 앱, 음성 대화 지원
  • Gemini — iOS / Android 앱, Google 앱과 통합

데스크톱과 기능 차이가 있을 수 있으나, 일상적인 질문·번역·요약 등은 모바일에서도 충분히 활용할 수 있습니다.

AI에게 시키면 안 되는 일은?

AI는 강력한 도구이지만, 다음은 반드시 주의해야 합니다.

  • 법률 자문 — AI의 답변은 참고 수준. 실제 법적 문제는 변호사에게
  • 의료 진단·처방 — 증상 설명은 도움이 되지만 의사 진료를 대체 불가
  • 금융 투자 결정 — 참고 자료로 활용하되 최종 결정은 전문가와 상의
  • 타인 개인정보 처리 — 동의 없이 제3자 정보 입력 금지
  • 사실 확인 없는 중요 내용 공유 — 환각 가능성을 항상 염두에 두기

"AI가 그렇게 말했다"는 것은 법적·의학적으로 면책 사유가 되지 않습니다.

안전하고 똑똑하게 쓰기

💡 핵심 원칙: AI는 도구입니다. 결과물의 최종 책임은 항상 사용하는 사람에게 있습니다. 신뢰하되 검증하세요.

좋은 습관

  • 중요한 사실·수치는 공식 출처에서 재확인한다
  • 민감한 개인정보는 AI에게 입력하지 않는다
  • "모르면 모른다고 알려줘" 라고 명시한다
  • 대화 기록 저장·학습 설정을 직접 확인한다
  • 법률·의료·금융은 전문가와 병행 활용한다
  • AI 답변을 그대로 퍼뜨리기 전에 한 번 더 검토한다
  • 프롬프트를 구체적으로 작성해 환각을 줄인다

피할 것

  • 주민등록번호, 카드번호, 비밀번호 입력 금지
  • 타인의 개인정보를 동의 없이 AI에 입력 금지
  • AI 답변을 검증 없이 중요 결정에 그대로 사용
  • 저작권 있는 콘텐츠를 통째로 재배포
  • 악성코드 작성·해킹·사기 등 불법 활동 요청
  • AI가 "확실하다"고 해도 사실 확인 없이 맹신
⚠️ 업무용 주의: 회사 기밀, 미공개 제품 정보, 고객 데이터는 외부 AI 서비스에 입력하기 전 회사 IT 정책을 반드시 확인하세요. 많은 기업이 허용 도구와 범위를 별도로 규정하고 있습니다.

프롬프트 해부도 — 5요소

좋은 프롬프트에는 공통적으로 다섯 가지 요소가 들어 있습니다. 모든 요소가 항상 필요한 건 아니지만, 더 많이 채울수록 결과가 좋아집니다.

아래는 5요소를 모두 갖춘 실제 프롬프트와 그 결과입니다.

너는 10년 경력의 한국 세무사야. 나는 프리랜서 개발자이고, 올해 처음으로 종합소득세를 신고할 예정이야. 합법적으로 세금을 줄이는 방법을 5가지로 정리해줘. 각 항목은 이름 + 한 줄 설명 + 실천 방법 순서로. 전문 용어는 쓰지 말고, 세금을 전혀 모르는 초보자도 이해할 수 있게 써줘.
📝 위 프롬프트의 실제 출력 예시

1. 경비 처리 (필요 경비 공제)
일을 하기 위해 쓴 돈은 수입에서 빼고 세금을 낼 수 있어요.
실천: 노트북·소프트웨어 구독·인터넷 요금 등 업무용 지출 영수증을 모두 보관하세요.

2. 국민연금·건강보험료 공제
프리랜서가 낸 4대 보험료는 전액 소득에서 공제됩니다.
실천: 납부 확인서를 홈택스에서 출력해 신고 시 첨부하세요.

3. 노란우산 공제 가입
중소기업 공제 상품으로, 납입액만큼 소득을 줄여 세금이 줄어요.
실천: 중소기업중앙회 홈페이지에서 가입, 월 최대 100만 원까지 납입 가능.

(이하 생략 — 실제로는 5가지 항목이 모두 출력됩니다)

팁: 5요소를 한 번에 다 쓰기 어렵다면, 역할 + 작업 + 출력 형식 세 가지만 먼저 챙겨보세요. 이것만으로도 결과가 크게 달라집니다.

좋은 프롬프트 7원칙

수십 번의 실험으로 검증된 7가지 원칙입니다. 각 원칙에는 나쁜 예와 좋은 예를 함께 수록했습니다.

① 구체성 — 무엇을, 누구를 위해, 얼마나

AI는 불분명한 부분을 자신의 가정으로 채웁니다. 모호한 단어 하나가 전혀 다른 결과를 만듭니다. "많이"보다 "5개", "짧게"보다 "200자 이내"가 훨씬 명확합니다.

✗ 나쁜 예마케팅 글 좀 써줘.
✓ 좋은 예30대 직장인 여성 대상, 스테인리스 텀블러 신제품(39,000원) 홍보용 인스타그램 게시글 3개를 써줘. 각 2~3문장, 해시태그 5개, 친근하고 감성적인 톤으로.
분량을 숫자로"몇 개", "몇 자", "몇 줄"로 정확히 표현
대상을 명시누가 읽을 글인지, 누구를 위한 답인지
목적을 밝히기이 글을 어디에 쓸 건지
범위를 좁히기한국 한정, 2026년 기준 등

② 맥락 제공 — 배경이 많을수록 정확

AI는 프롬프트 안에서만 상황을 파악합니다. 에러 메시지, 코드 원문, 현재 상황 등 관련 정보를 아끼지 말고 함께 주세요. 특히 코딩·문서 수정·비즈니스 질문에서 맥락의 효과가 두드러집니다.

✗ 나쁜 예이 코드 왜 안 돼?
✓ 좋은 예파이썬 3.11에서 아래 코드를 실행하면 'KeyError: age'가 발생해. 원인을 설명하고 수정된 전체 코드를 보여줘. 코드: user = {'name': 'Alice'} / print(user['age'])
📝 입력 (맥락 충분히 제공)
파이썬 3.11. 딕셔너리에서 없는 키를 접근하면 KeyError가 난다고 들었어. 아래 코드에서 발생하는 문제를 설명하고, .get() 메서드를 써서 안전하게 수정해줘. 코드: user = {'name': 'Alice'} print(user['age'])
✨ 출력

문제 원인: user 딕셔너리에 'age' 키가 존재하지 않아 KeyError가 발생합니다.

수정된 코드:
user = {'name': 'Alice'}
print(user.get('age', '정보 없음'))

설명: .get(키, 기본값)은 키가 없어도 에러 대신 기본값을 반환합니다. 기본값을 지정하지 않으면 None이 반환됩니다.

③ 예시 제공 — 결과물 샘플을 보여주기 (Few-shot)

"이런 스타일로 써줘"라는 말보다 실제 예시 1~2개를 보여주는 것이 훨씬 효과적입니다. AI는 예시를 패턴으로 학습하여 동일한 스타일을 적용합니다.

✗ 나쁜 예친환경 세제 브랜드 이름 지어줘.
✓ 좋은 예친환경 세제 브랜드 이름 5개를 지어줘. 스타일 예시: '에코밤(EcoBalm)', '그린드롭(GreenDrop)' — 한글+영어 병기, 발음 쉬운 두 음절, 자연을 연상시키는 단어 사용.

④ 역할 지정 — 전문가 모자를 씌우기

"너는 [역할]이야"로 시작하는 것만으로 AI의 응답 관점과 전문성이 달라집니다. 같은 질문이어도 의사, 변호사, 교사로 역할을 바꾸면 전혀 다른 각도의 답변이 나옵니다.

✗ 나쁜 예세금 어떻게 줄여?
✓ 좋은 예너는 10년 경력의 한국 세무사야. 프리랜서 개발자가 종합소득세를 합법적으로 줄이는 방법 5가지를, 세금 초보자도 이해할 수 있게 설명해줘.

역할 지정 예시 모음

  • 너는 고등학교 수학 선생님이야.
  • 너는 20년 경력 편집장이야.
  • 너는 스타트업 마케터이야.
  • 너는 한국어 원어민 번역가야.
  • 너는 친절한 헬스 트레이너야.

역할 지정 주의사항

  • 의료·법률·금융 답변은 전문가 상담 대체 불가
  • 역할을 줘도 AI가 틀릴 수 있음 — 사실 확인 필수
  • 너무 과장된 역할은 오히려 결과를 왜곡
  • 역할과 작업이 모순되지 않게 설정

⑤ 단계적 사고 유도 — "차근차근 생각해줘"

복잡한 문제에 "단계별로 생각해줘", "먼저 분석하고 그 다음 답해줘" 같은 표현을 추가하면 AI가 중간 추론 과정을 거쳐 더 정확한 답을 냅니다. 특히 수학·논리·분석 문제에 효과적입니다.

✗ 나쁜 예사과 12개를 3명에게 나누고 2개 더 받으면 한 명당 몇 개?
✓ 좋은 예다음 문제를 단계별 풀이 과정을 보여주며 풀어줘. 마지막 줄에 정답만 따로. 문제: 사과 12개를 3명에게 똑같이 나눈 뒤 각자 2개씩 더 받으면 한 명당 몇 개?
📝 입력 — 단계적 사고 요청
아래 비즈니스 시나리오를 분석하고, 문제점 → 원인 → 해결책 순서로 차근차근 분석해줘. 마지막에 핵심 요약 1~2문장으로 마무리해줘. 시나리오: 온라인 쇼핑몰의 장바구니 이탈률이 지난달 대비 15% 상승했고, 신규 가입자는 줄었다.
✨ 출력

1단계 — 문제 파악: 장바구니 이탈률 15% 상승 + 신규 가입자 감소는 구매 전환과 유입 모두 문제가 있음을 시사합니다.

2단계 — 원인 추정: ① 결제 프로세스 복잡/오류, ② 배송비·가격 경쟁력 저하, ③ 랜딩 페이지 UX 문제, ④ 경쟁사 프로모션에 유입 감소 가능성.

3단계 — 해결책: 결제 단계 간소화(1페이지 결제), 무료 배송 기준 재검토, 신규 유입 광고 채널 A/B 테스트, 장바구니 리타게팅 메일 발송.

핵심 요약: 이탈률 급등은 UX 또는 가격 경쟁력 이슈일 가능성이 높고, 결제 흐름 개선과 리타게팅이 가장 빠른 해결책입니다.

⑥ 출력 형식 지정 — 표·목록·글자 수

AI는 형식을 지정하지 않으면 자신이 편한 방식으로 씁니다. 원하는 형식(표, 번호 목록, JSON, 마크다운 등)과 분량을 명시하면 바로 사용 가능한 형태로 나옵니다.

✗ 나쁜 예노트북 추천해줘.
✓ 좋은 예2026년 기준 100만 원 이하 노트북 5종을 표로 정리해줘. 열: [모델명 | CPU | RAM | 가격 | 추천 대상]. 표 아래에 '영상 편집용이라면 어떤 걸 선택할지' 이유와 함께 한 줄로 추가해줘.
자주 쓰이는 형식 지정 표현: "표로 정리해줘" / "번호 목록으로" / "마크다운으로 작성해줘" / "JSON 형식으로" / "500자 이내로 요약해줘" / "한 문단만"

⑦ 제약과 톤 지정 — 하지 말 것·분량·말투

금지 사항을 명시하면 AI가 불필요한 부분을 제외하고 핵심에 집중합니다. 말투(존댓말/반말, 전문적/친근함)도 지정할 수 있습니다.

✗ 나쁜 예회사 사과문 써줘.
✓ 좋은 예배송 지연 사과 문자를 써줘. 조건: 200자 이내, 정중하지만 과하지 않은 존댓말, 변명 금지, 다음 구매 10% 할인 보상 한 문장 포함. 서명은 '[회사명]'으로.
제약 표현 예시: "전문 용어 사용 금지" / "추측하지 말고 모르면 솔직히 말해줘" / "영어 단어를 섞지 마" / "광고성 표현 금지" / "결론 먼저, 이유는 그 다음"

한 단계 위 기법

7원칙을 익혔다면, 아래 6가지 고급 기법으로 결과를 한층 더 정교하게 만들 수 있습니다.

Few-shot — 예시를 여러 개 제공하기

출력 형식이 복잡하거나 특정 스타일을 정확히 재현하고 싶을 때, 예시를 2~3개 연속으로 보여주면 AI가 패턴을 학습하여 동일한 형식으로 나머지를 생성합니다.

아래 형식으로 제품 설명 카드를 3개 만들어줘. 예시 1: 제품명: 에코텀블러 500ml 한 줄 설명: 지구를 아끼는 당신을 위한 선택 가격: 39,000원 예시 2: 제품명: 슬림 무선 충전기 한 줄 설명: 케이블 없는 깔끔한 책상의 시작 가격: 28,000원 이제 같은 형식으로 [제품 A], [제품 B], [제품 C]의 설명 카드를 만들어줘.

Chain-of-Thought — 차근차근 생각하게 하기

"단계별로 생각해줘" 또는 "먼저 전제를 나열하고 그 다음 결론을 내려줘"처럼 추론 과정을 요구하면 특히 수학·논리·전략 분석에서 정확도가 높아집니다.

다음 의사결정 문제를 분석할 때, 아래 단계로 차근차근 생각해줘. 1단계: 핵심 목표와 제약 조건 파악 2단계: 선택지별 장단점 나열 3단계: 각 선택지의 위험도 평가 4단계: 최종 추천 및 이유 문제: [여기에 의사결정 상황 입력]

페르소나 고정 — 대화 내내 역할 유지시키기

긴 대화에서 AI가 역할에서 벗어날 수 있습니다. 처음에 페르소나를 명확히 설정하고, 중간에 벗어나면 "처음에 말한 역할로 계속해줘"라고 다시 상기시키세요.

이번 대화에서 너는 '짧은 문장을 좋아하는 미니멀리스트 카피라이터'야. 내가 긴 문장을 주면 짧고 강렬하게 바꿔줘. 역할에서 벗어나지 말고, 추가 설명도 하지 마. 오직 수정된 카피만 출력해. 첫 번째 문장: '저희 제품은 고객 여러분께서 일상에서 불편하게 느끼셨던 여러 문제들을 해결해 드리기 위해 최선을 다해 만들었습니다.'

반복 개선 (Iteration) — 대화로 다듬기

처음 결과가 마음에 들지 않아도 괜찮습니다. "더 짧게", "톤을 바꿔줘", "두 번째 단락을 다시 써줘"처럼 구체적인 수정 요청을 이어나가면 점점 원하는 결과에 가까워집니다.

"더 짧게 줄여줘"분량 조정
"톤을 더 가볍게"말투·느낌 변경
"3번 항목만 다시"부분 재생성
"다른 버전 3개"선택지 확보

자기 검증 — "스스로 검토해줘"

AI의 답변 뒤에 "방금 답변에서 틀린 부분이나 보완할 점이 있으면 알려줘"라고 추가하면 AI가 스스로 오류를 점검합니다. 특히 사실 확인이 중요한 분야에서 유용합니다.

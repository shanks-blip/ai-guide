활용 사다리 — 어디까지 왔나요?

업데이트 · 2026-06-19

심화로 갈수록 "내가 직접 한다"에서 "AI가 대신 한다"로 무게가 옮겨갑니다. 아래 다이어그램은 4단계 여정을 한눈에 보여줍니다.

단계가 높아질수록 설정 비용은 오르지만 반복 비용은 0에 가까워집니다. 처음에는 1단계로 시작해 익숙해지면 조금씩 올라오세요.

① 대화로 쓰기

질문·생성·분석·요약. 가장 기본이자 지금도 가장 많이 쓰는 단계. 이 가이드의 시작하기와 프롬프트 작성법이 여기에 해당합니다.

② 도구 연결

파일·웹 검색·커넥터(MCP)를 붙이면 AI가 실제 자료에 접근해 더 정확한 결과를 냅니다. Claude 데스크톱의 MCP 연결이 대표적입니다.

③ 자동 실행

예약 작업·cron·서버리스로 정해진 시간에 AI가 스스로 반복 수행합니다. 매일 아침 브리핑, 주간 리포트 등이 여기에 속합니다.

④ 에이전트·API

여러 단계를 자율 계획·실행하거나, 내 앱·서비스 코드 안에 AI를 내장합니다. SDK를 쓰면 완전히 맞춤형 워크플로를 구성할 수 있습니다.

API 기초 — 내 코드에 AI 넣기 고급

업데이트 · 2026-06-19

API(Application Programming Interface)를 쓰면 챗 화면 밖, 내 프로그램·서비스 안에서 AI 모델을 직접 호출할 수 있습니다. 대화 UI 없이도 문서를 요약하거나, 응답을 데이터베이스에 저장하거나, 파이프라인에 연결할 수 있게 됩니다.

실제로 돌려보려면 — 개발자 가이드. 이 페이지는 큰 그림·개념 중심입니다. 가상환경·.env 설정과 복붙해서 끝까지 돌아가는 전체 코드(멀티턴·스트리밍·구조화 출력·재시도)는 개발자 가이드 → API·SDK 심화에 있습니다.

공통 흐름 — 세 단계

API 키 발급

공식 콘솔에서 계정을 만들고 키를 생성합니다. Anthropic Console ↗ 또는 OpenAI Platform ↗에서 발급하세요.

환경변수에 키 저장

키를 코드에 직접 적지 않고 .env 파일이나 시스템 환경변수에 보관합니다. ANTHROPIC_API_KEY, OPENAI_API_KEY 형식이 관례입니다.

SDK로 첫 호출

공식 Python·Node.js SDK를 설치하고 메시지를 보냅니다. 응답 객체에서 텍스트를 꺼내 쓰면 됩니다.

코드 예시

# pip install anthropic import anthropic client = anthropic.Anthropic() # ANTHROPIC_API_KEY 환경변수 자동 참조 message = client.messages.create( model='claude-opus-4-8', max_tokens=1024, messages=[ {'role': 'user', 'content': '세 줄로 파이썬을 설명해줘.'} ] ) print(message.content[0].text)
OpenAI Python: pip install openai 후 from openai import OpenAI를 쓰면 구조가 거의 동일합니다. 모델명과 환경변수만 다릅니다(OPENAI_API_KEY).
// npm install @anthropic-ai/sdk import Anthropic from '@anthropic-ai/sdk'; const client = new Anthropic(); // ANTHROPIC_API_KEY 환경변수 자동 참조 const message = await client.messages.create({ model: 'claude-opus-4-8', max_tokens: 1024, messages: [ { role: 'user', content: '세 줄로 파이썬을 설명해줘.' } ] }); console.log(message.content[0].text);
OpenAI Node.js: npm install openai 후 import OpenAI from 'openai'. 메서드 이름은 chat.completions.create이고 모델 파라미터만 다릅니다.
보안 필수: API 키는 비밀번호와 같습니다. 코드에 직접 적거나 GitHub에 올리면 즉시 유출됩니다. .env 파일과 .gitignore를 꼭 함께 쓰세요. 사용량에 따라 비용이 발생하니 월 한도(Budget)를 설정해 두는 것도 좋습니다.

MCP · 커넥터 — AI를 외부 도구와 잇기 고급

업데이트 · 2026-06-19

MCP(Model Context Protocol)는 AI와 외부 도구·데이터 소스를 안전하게 연결하는 오픈 표준입니다. 한 번 표준을 따르면 같은 커넥터 서버를 여러 AI에서 재사용할 수 있습니다. 파일시스템·Git·Slack·Google Drive·데이터베이스 등 커넥터를 붙이면 AI가 그 자료를 직접 읽고 조작할 수 있게 됩니다.

직접 서버 만들기. 남이 만든 서버 연결을 넘어 내 함수·데이터를 노출하는 MCP 서버 제작은 개발자 가이드 → MCP 서버 만들기(FastMCP)에서 단계별로.

Claude와 OpenAI에서 MCP 쓰는 법

🟠 Claude (데스크톱 / Claude Code) 설정 파일의 mcpServers 섹션에 서버 경로·명령어 등록. Claude Code는 claude mcp add 명령으로 바로 추가.
🟢 OpenAI (Agents SDK) Agents SDK에서 MCP 서버를 Tool로 연결. Codex CLI도 MCP 도구를 지원합니다.
공식 서버 모음 modelcontextprotocol/servers ↗ — 파일·Git·Slack·Postgres 등 레퍼런스 서버 수록.
MCP 공식 문서 modelcontextprotocol.io ↗ — 표준 스펙·구현 가이드·SDK 문서.
팁: 처음 MCP를 써 본다면 파일시스템 서버부터 시작하세요. 로컬 폴더를 AI에 연결해 파일을 읽고 요약하는 경험이 가장 직관적입니다.

함수 호출 (Function Calling) 고급

업데이트 · 2026-06-21

함수 호출은 AI가 텍스트 생성에 그치지 않고 실제 행동(조회·예약·주문·전송)을 할 수 있게 하는 핵심 기술입니다. AI가 어떤 함수를 어떤 인자로 호출할지 JSON으로 결정하면, 내 코드가 그것을 실행하고 결과를 다시 AI에게 돌려줍니다.

실행되는 예제. tool use 엔드투엔드(모델이 JSON으로 호출 → 내 코드 실행 → 결과 반환) 루프 코드는 개발자 가이드 → 에이전트·구조화 출력에 있습니다.

함수 호출로 가능한 것들

예약·일정 관리

캘린더 API를 연결해 회의·예약을 대화만으로 생성·수정·삭제.

실시간 조회

날씨·환율·재고·주문 상태 등 외부 API 데이터를 즉시 가져와 답변.

주문·결제 처리

이커머스 백엔드와 연결해 장바구니·결제·취소를 자연어로 처리.

DB 쓰기·읽기

데이터베이스 CRUD를 AI가 직접 호출해 데이터 입력·수정을 자동화.

함수 호출, 4단계로 직접 보기

1
도구(함수) 정의. 이름·설명·인자를 JSON 스키마로 적어 모델에게 알려줍니다. 설명이 구체적일수록 모델이 정확히 고릅니다.
2
모델이 호출 결정. 사용자 요청을 보고 ‘어떤 함수를 어떤 인자로’ 부를지 JSON으로 돌려줍니다(실행은 아직 안 함).
3
내 코드가 실행. 그 JSON을 받아 실제 API·DB를 호출하고 결과를 받습니다.
4
결과를 다시 모델에게. 함수 결과를 전달하면 모델이 사람이 읽을 답으로 마무리합니다.
{ "name": "get_weather", "description": "도시의 현재 날씨를 조회한다", "parameters": { "type": "object", "properties": { "city": { "type": "string", "description": "도시 이름" } }, "required": ["city"] } }
사용자

“서울 날씨 어때?”

모델의 함수 호출(JSON)

get_weather(city="서울") → 내 코드가 실행 후 결과 반환 → “서울은 맑고 24도예요.”

막히는 지점해결
모델이 함수를 안 부르고 말로만 답한다함수 description을 구체적으로, 시스템 프롬프트에 ‘언제 이 함수를 써라’를 명시
엉뚱한 인자를 채운다각 인자에 description·형식·예시를 적고, required로 필수값 강제
여러 함수 중 잘못 고른다함수 수를 꼭 필요한 것으로 줄이고 이름을 역할이 드러나게(get_/create_ 등)
실행 결과를 안 돌려줘 흐름이 끊긴다tool 결과를 반드시 다시 모델에 전달하는 루프를 코드로 구현(개발자 가이드 참고)
📖 OpenAI — Function calling 가이드 (공식) ↗함수 정의·호출 흐름·베스트 프랙티스 📖 Claude — Tool use 개요 (공식) ↗client/server tool, tool_use·tool_result 루프

자동화 · 예약 작업 고급

업데이트 · 2026-06-19

매번 직접 시키지 않아도, 정해진 시간에 AI가 스스로 일하게 만들 수 있습니다. 이 가이드 사이트의 "오늘의 AI" 피드도 같은 원리로 매일 자동 갱신됩니다.

Claude Cowork /schedule

대화창에 /schedule을 입력하면 반복 작업을 예약할 수 있습니다. Claude 데스크톱 앱이 켜져 있을 때 실행됩니다.

Claude Code Routines

Claude Code에서 Routines를 설정하면 클라우드 인프라에서 실행되어 로컬 PC가 꺼져 있어도 작동합니다.

Cron · 서버리스

서버나 AWS Lambda·Vercel Functions에서 cron 표현식으로 API를 정기 호출. 완전 자동화의 가장 범용적인 방법.

n8n · Zapier · Make

코드 없이 워크플로 자동화 도구로 AI 호출을 트리거에 연결. 이메일 수신, 폼 제출, 캘린더 이벤트 등과 손쉽게 연동.

자동화 프롬프트 예시

매일 아침 8시에 내 받은편지함에서 어제 온 메일을 훑어보고, 중요도 순으로 3개를 골라 각각 한 줄 요약과 답장 초안을 만들어줘.
매주 금요일 오후 5시에 이번 주 완료한 작업 목록을 정리하고, 다음 주 우선순위 Top 3를 제안하는 주간 리포트를 만들어줘.
이 사이트의 자동화: "오늘의 AI" 피드(updates.html)는 자동화된 스크립트가 매일 최신 AI 뉴스·논문·GitHub 소식을 수집해 data/updates.js?v=20260622234347를 갱신하는 방식으로 동작합니다. 여러분도 같은 원리로 자신만의 자동 뉴스레터나 리포트 봇을 만들 수 있습니다.

커넥터 · 확장 프로그램 연동 고급

업데이트 · 2026-06-19

커넥터와 브라우저 확장으로 AI를 내 도구·웹·업무 흐름에 직접 붙이면, '복사-붙여넣기' 없이 실제 데이터를 다루게 됩니다. 무엇을 어떻게 연결하고 어떤 업무에 쓰는지 정리했습니다.

커넥터(Connectors) — 내 앱·데이터에 연결

커넥터는 AI를 Google Drive·Slack·Notion·GitHub 같은 외부 서비스에 안전하게 연결합니다(권한은 내가 승인하고 언제든 해제). 내부적으로 MCP 표준을 활용합니다.

Google Drive내 문서를 근거로 답변·요약
Slack스레드 요약·답장 초안·알림
Notion문서·DB 조회·정리
GitHub코드·이슈·PR 질의

연결 (Claude)

설정 → 커넥터(Connectors)에서 서비스를 연결하고 권한을 승인합니다. Cowork에서는 작업 중 필요한 커넥터를 바로 쓸 수 있습니다. (커넥터 둘러보기 ↗)

연결 (ChatGPT)

설정 → Connectors/Apps에서 Google Drive·GitHub 등을 연결합니다. (예: GitHub 연결 ↗)

권한 최소화

필요한 폴더·저장소만 허용하고, 민감 데이터는 신중히. 권한은 설정에서 언제든 회수하세요.

확장 프로그램 — 브라우저·업무 앱에 붙이기

브라우저 확장을 쓰면 AI가 지금 보고 있는 웹페이지를 읽고, 요약·표 추출·양식 작성을 그 자리에서 도와줍니다. 메신저·오피스 연동으로 업무 도구 안에서 바로 부를 수도 있습니다.

Claude in Chrome현재 페이지 읽기·작업 보조 (소개 ↗)
Claude for Slack · M365메신저·오피스에서 Claude 호출
ChatGPT 브라우징·에이전트웹을 탐색하며 작업 수행
보안 주의: 확장은 페이지 내용을 읽습니다. 로그인된 민감 사이트·사내 시스템에서는 권한과 동작을 꼭 확인하고, 신뢰할 수 없는 페이지의 지시(프롬프트 인젝션)는 따르지 않도록 주의하세요.

노코드 자동화 — Zapier·Make·n8n으로 AI 끼우기 고급

추가 · 2026-06-22

목표. 코드 한 줄 없이, 평소 쓰는 앱들(폼·메일·시트·메신저) 사이에 AI 한 단계를 끼워 반복 업무를 자동으로 처리합니다.  이럴 때. “폼이 들어오면 요약해서 슬랙에 알려줘”, “메일이 오면 분류해 시트에 정리해줘”처럼 트리거 → 처리 → 액션이 반복되는 일에 활용하면 좋아요.

대표 도구는 셋입니다. Zapier는 가장 쉽고 연결 가능한 앱이 많아 입문에 좋고, Make는 화면에서 흐름을 그리듯 시각적으로 짜며, n8n은 더 세밀한 제어와 셀프호스팅까지 됩니다. 공통점은 트리거→액션 구조 사이에 AI 스텝을 끼운다는 점입니다.

1
무료로 가입하고 템플릿을 둘러봅니다. 비슷한 자동화를 골라 시작하면 빠릅니다.
2
트리거 연결. “새 폼 응답”, “새 메일”, “매일 아침 9시”처럼 시작 신호가 될 앱·이벤트를 고릅니다.
3
AI 스텝 추가. Zapier의 ‘AI by Zapier’, Make의 OpenAI·AI Toolkit 모듈, n8n의 AI Agent 노드를 끼워 요약·분류·추출을 맡깁니다.
4
필드 매핑. 앞 단계의 값(제목·본문 등)을 AI 입력에, AI 결과를 다음 액션(메일·시트·슬랙)에 연결합니다.
5
테스트 → 게시. 한 건으로 미리보기를 확인하고, 문제없으면 켜서 자동 실행합니다.

예를 들어 ‘AI by Zapier’는 Zap 안에 직접 만드는 AI 액션으로, 프롬프트(무엇을 시킬지)·입력 필드(맥락)·출력 필드(결과 구조) 세 영역으로 동작합니다. 자기 OpenAI·Anthropic·Gemini 계정을 연결(BYOK)하거나 Zapier 내장 연결을 그대로 쓸 수 있습니다. AI 스텝에 넣을 지시는 이렇게 적습니다.

아래 [문의 내용]을 한국어 3줄로 요약하고, ‘환불·버그·기능요청·기타’ 중 하나로 분류해줘. 결과는 ‘요약’과 ‘분류’ 두 항목으로 나눠서 줘.
자동화 시나리오

고객 문의 폼 제출(트리거) → AI 스텝이 요약·분류 → 슬랙 #cs 채널에 카드로 전송(액션)

담당자가 보는 것

“[환불] 결제 두 번됨 — 어제 12,000원 중복 청구, 환불 요청”처럼 한 줄로 정리된 알림

또 다른 예

매일 아침 9시(스케줄 트리거) → 신규 가입 시트를 AI가 요약 → 팀 메일로 일일 리포트

왜 좋은가

사람이 매일 시트를 열어보지 않아도, 정해진 시각에 핵심만 추려 받아볼 수 있음

막히는 지점은 대개 정해져 있습니다. 미리 알아두면 헤매지 않습니다.

증상원인해결
AI 스텝이 이전 실행을 기억 못 하거나 웹 주소(URL) 내용을 못 읽음AI by Zapier는 실행할 때마다 지식이 초기화되고, 웹·URL에서 정보를 검색·추출하지 못함(공식 문서 명시)필요한 맥락은 ‘입력 필드’로 직접 넣거나 파일·문서를 지식 소스로 붙여 제공
무료로 만들었는데 AI 모듈이 안 돌아감AI by Zapier는 무료 플랜 미지원(Professional 이상). Make AI Toolkit 무료는 주당 입력 토큰 한도가 있음유료 여부 확인. Make는 키 없이 되는 ‘Simple text prompt’·자체 AI로 먼저 테스트
OpenAI 등 외부 AI 연결이 실패API 키·조직 정보 미입력 또는 계정 크레딧·권한 부족제공사에서 키 발급 후 입력·결제 설정. 키가 부담되면 내장 연결부터 시작
  • 자동화의 트리거·액션을 한 문장으로 적어봤는가 (“X가 오면 → Y를 한다”)
  • AI 스텝에 넘기는 입력에 민감정보가 없는지 확인했는가 (→ 안전하게 쓰기)
  • 한 건으로 테스트한 뒤 켰는가, 무료 한도·과금을 확인했는가
  • AI 결과를 그대로 내보내기 전에 검증이 필요한 업무인가 (→ AI 답 검증)

도구·플랜은 자주 바뀝니다. 무료 한도, ‘AI by Zapier’의 지원 플랜(현재 Professional 이상), Make·n8n의 AI 모듈 구성은 업데이트될 수 있으니 가격·한도는 공식 문서에서 확인하세요. AI 스텝의 결과도 검증이 필요한 업무라면 사람이 확인한 뒤 내보내세요.

업데이트 · 2026-06-19
목표. '그때그때 시키는' 바이브코딩에서, AI가 스스로 돌고 스스로 검증하는 환경(하네스)을 갖춘 에이전틱 코딩으로 올라섭니다.  핵심. 똑똑한 모델보다 '모델이 일하기 좋은 환경'이 결과를 가릅니다.
바이브코딩한 번에 하나씩 시키고 결과를 사람이 일일이 확인. 빠르지만 규모가 커지면 흔들림.
에이전틱 코딩테스트·린트·문서·규칙이 갖춰진 환경에서 AI가 계획→실행→검증을 반복. 사람은 방향과 승인만.

1) 하네스를 갖추는 단계

자동 검증 장치

테스트·린트·타입체크처럼 "맞는지 기계가 판단"하는 장치를 둡니다. AI가 스스로 통과 여부를 알 수 있어야 합니다.

규칙·메모리·스킬

프로젝트 규칙(코딩 가이드), 작업 기억(메모리), 반복 작업(스킬)을 미리 정의해 매번 설명하지 않게 합니다.

작은 단위 + 승인 지점

큰 목표를 작은 작업으로 쪼개고, 위험 작업 전에는 사람 승인을 받게 합니다.

이 프로젝트를 에이전틱하게 작업할 수 있게 환경을 점검해줘. - 자동 검증(테스트·린트)이 충분한지 - AI가 따를 규칙 파일이 있는지 - 작업을 작은 단위로 어떻게 쪼갤지 부족한 부분과, 무엇부터 갖추면 좋을지 우선순위로 알려줘.

우리 레포에 맞는 '코딩 규칙 파일(가이드라인)' 초안을 만들어줘. 폴더 구조, 네이밍, 테스트 작성 기준, 커밋 규칙을 포함해서.

업데이트 · 2026-06-19
목표. 똑똑한 AI를 쓰는 것만큼 중요한 'AI가 날아다닐 수 있는 환경(컨텍스트)'을 갖춥니다. codex·claude code 같은 도구의 성능을 끌어올리는 토대입니다.

1) 좋은 환경 체크리스트

  • 명확한 프로젝트 구조 — 폴더·파일이 일관되어 AI가 길을 잃지 않음
  • 규칙 파일(가이드라인·컨벤션) — 매번 설명하지 않아도 됨
  • 충분한 테스트 — AI가 스스로 맞는지 확인할 근거
  • 작은 단위 작업 — 한 번에 거대한 작업 대신 쪼개서
  • 깔끔한 컨텍스트 — 불필요한 파일·정보로 혼선 주지 않기
흔한 실수좋은 환경
"알아서 다 해줘" 한 방에 던지기목표를 작은 단계로 쪼개 맡기기
규칙·맥락을 매번 말로 설명규칙 파일·메모리로 한 번만 정의
검증 없이 결과 신뢰테스트·린트로 자동 검증
거대한 파일·잡다한 컨텍스트관련 파일만, 구조를 단순하게

2) 바로 적용하는 프롬프트

내 프로젝트가 AI(코덱스·Claude Code)가 일하기 좋은 환경인지 진단해줘. - 구조·규칙 파일·테스트·작업 단위·컨텍스트 정리 관점으로 점검 - 점수와 함께, 가장 효과 큰 개선 3가지를 우선순위로 알려줘

무엇을 맡길지 정하기

설치

공식 저장소/문서의 안내대로 설치합니다. 보통 서버(VPS)나 로컬 환경에 올립니다. (예: hermes-agent.nousresearch.com ↗)

세팅

모델·API 키·접근 권한을 설정합니다. 키는 환경변수로 안전하게.

작게 사용하며 신뢰 쌓기

처음엔 읽기·요약 같은 안전한 작업부터. 잘 돌면 범위를 넓힙니다. 로그를 보며 동작을 확인하세요.

2) 운영 전 안전 체크

위험대비
키·토큰 노출코드에 넣지 말고 환경변수·시크릿으로

비용 폭주호출·예산 한도와 알림 설정
신뢰할 수 없는 코드공식·검증된 저장소만, 설치 전 README·권한 확인

3) 이어서 할 후속 질문
